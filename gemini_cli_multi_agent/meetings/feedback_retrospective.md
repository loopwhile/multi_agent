# Meeting: Feedback Retrospective

## 목적

배포 후 **사용자 피드백, 운영 결과, 프로세스 개선점**을 회고한다.

## 참석 역할

- another_me (진행)
- growth_feedback_agent (피드백 보고)
- operations_workflow_agent (운영 결과 보고)
- product_scope_agent (다음 스코프 반영)

## 트리거

- 배포 후 일정 기간 경과
- 핵심 지표에 유의미한 변화 발생
- 사용자 피드백 임계치 초과

## 아젠다

### 1. 운영 결과 보고 (5분)

- 배포 후 시스템 안정성
- 에러율 / 성능 지표
- 인프라 이슈

### 2. 사용자 피드백 분석 (10분)

- 피드백 카테고리별 요약
- 주요 패턴 / 반복 이슈
- 긍정적 피드백 / 부정적 피드백

### 3. 지표 리뷰 (5분)

- 핵심 지표 변화 (배포 전후 비교)
- 예상 대비 실제 결과

### 4. 개선 제안 논의 (10분)

- growth_feedback_agent의 Improvement Proposals 검토
- 우선순위 논의
- 스코프 반영 여부 결정

### 5. 프로세스 회고 (5분)

- 이번 사이클에서 잘 된 것
- 개선이 필요한 것
- 워크플로우 조정 필요 사항

## 출력

```markdown
## Retrospective Minutes

### 날짜: YYYY-MM-DD
### 대상: {프로젝트명 / Work Order}

### 핵심 지표
| 지표 | 배포 전 | 배포 후 | 변화 |
|------|---------|---------|------|
| {지표} | {값} | {값} | {변화} |

### 채택된 개선 제안
- {제안 1} → product_scope_agent: Work Order 생성

### 프로세스 개선
- {개선 1}

### Action Items
- [ ] {액션 1} — 담당: {역할}
```
