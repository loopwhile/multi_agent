# Adapter Spec

## 1. 목적

이 문서는 AI 실행 도구가 **Agent OS 공용 코어를 어떻게 로드하고 실행할지** 안내하는 `adapter` 문서의 표준 규격을 정의한다.

**중요 전제**:
- adapter는 운영 규약의 원천이 아니다.
- 공용 코어(`docs/`, `ai_prompts/roles/` 등)의 내용을 복제하지 않는다.
- 오직 "이 도구의 환경에서 어떻게 시작할 것인가"만 다룬다.

## 2. 필수 포함 섹션

모든 adapter(예: `antigravity.md`)는 아래 형식을 따라야 한다.

```markdown
# Adapter: {도구명}

## 1. 목적
이 문서가 어떤 실행 환경을 위한 매핑 가이드인지 명시.

## 2. 읽기 순서 (초기화)
이 도구에서 새로운 대화/세션을 시작할 때 맨 처음 읽어야 하는 필수 공용 문서 목록.
예) docs/00_overview.md → docs/03_rules.md → 이 adapter

## 3. 권장 실행 흐름
이 도구의 인터페이스 특성에 맞춘 권장 작업 방식. (예: 터미널 기반이면 CLI 명령어 위주, 챗 기반이면 대화 위주)

## 4. Work Order 생성 & 진행 방식
도구 내에서 `ai_prompts/tasks/_template.md`를 어떻게 활용하여 Work Order를 시작할지 안내.

## 5. 필수 준수 경로
출력 생성 전 반드시 참고해야 하는 공통 규약 문서 경로 (절대 복제 금지).
- Output Contract 경로
- Verification Gate 경로

## 6. 제한사항 / 워크어라운드
해당 도구만이 가지는 태생적 한계(예: 브라우저 접근 불가, 컨텍스트 윈도우 짧음 등)와 그 해결책.

## 7. 금지사항
도구 환경에 상관없이 절대 해서는 안 될 행동 명시.
```

## 3. 위반 감지 기준
다음의 내용이 adapter에 포함되어 있다면 `docs/08_tool_adapter_policy.md` 위반이다.
- 역할(Role)의 세부 책임과 권한 서술
- 검증 기준(Checklist) 본문 복제
- 프로젝트 특수 정보(API Key, 엔드포인트) 하드코딩
