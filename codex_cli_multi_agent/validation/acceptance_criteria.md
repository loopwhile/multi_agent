# Acceptance Criteria Guide

## 목적

Work Order의 **수락 기준(Acceptance Criteria) 작성 원칙과 가이드**를 정의한다.  
수락 기준은 Verification Gate 통과의 핵심 판단 근거다.

---

## 좋은 수락 기준의 특성

| 특성 | 설명 | 나쁜 예 | 좋은 예 |
|------|------|---------|---------|
| **검증 가능** | Yes/No로 판정 가능 | "성능이 좋아야 함" | "API 응답 시간 < 200ms (p95)" |
| **구체적** | 모호함 없음 | "에러 처리가 적절함" | "401, 403, 500 에러에 대해 구조화된 JSON 응답 반환" |
| **독립적** | 다른 기준과 독립 판정 가능 | "전체적으로 잘 동작함" | "로그인 API가 유효한 JWT를 반환함" |
| **범위 한정** | 이 WO의 스코프 내 | "모든 API 보안 검증" | "이 WO에서 추가한 2개 엔드포인트의 인증 검증" |

---

## 작성 형식

```markdown
## 수락 기준

- [ ] {주어} + {동작} + {기대 결과}
- [ ] {주어} + {조건} + {기대 동작}
```

### 예시

```markdown
## 수락 기준

- [ ] 로그인 API가 유효한 credentials로 호출되면 JWT 토큰을 반환한다
- [ ] 로그인 API가 잘못된 credentials로 호출되면 401 응답을 반환한다
- [ ] JWT 토큰의 유효 기간이 24시간으로 설정되어 있다
- [ ] 빌드가 에러 없이 성공한다 (`npm run build` exit code 0)
- [ ] 기존 테스트가 모두 통과한다
```

## 유형별 기준 가이드

### 코드 기능

- 정상 경로 동작
- 에러 / 예외 처리
- 엣지 케이스 (빈 입력, 경계값, 대량 데이터)
- 빌드 성공
- 테스트 통과

### 설계

- 설계 문서 완성도
- ADR 존재 여부
- 영향 범위 식별
- builder가 바로 구현 시작 가능 여부

### 배포

- 배포 성공
- Smoke test 통과
- 모니터링 정상
- 롤백 가능 확인
