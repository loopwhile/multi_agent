# Quality Checklist

## 목적

모든 에이전트 출력과 Work Order 결과물에 적용되는 **품질 검증 체크리스트**.  
`quality_verification_agent`가 검증 시 이 체크리스트를 기준으로 판정한다.

---

## 공통 체크리스트

모든 유형의 작업에 적용된다.

- [ ] Output Contract 형식을 준수하는가
- [ ] Evidence가 포함되어 있는가
- [ ] Evidence가 작업 내용과 일치하는가
- [ ] 수락 기준(Acceptance Criteria)이 모두 충족되었는가
- [ ] 역할 범위를 벗어난 판단이 포함되지 않았는가
- [ ] 프로젝트 특수사항이 역할 프롬프트에 하드코딩되지 않았는가

---

## 코드 작업 체크리스트

코드 구현/수정 작업에 추가 적용된다.

### 기능성
- [ ] 요구된 기능이 구현되었는가
- [ ] 엣지 케이스가 처리되었는가
- [ ] 에러 핸들링이 적절한가

### 빌드 & 테스트
- [ ] 빌드가 성공하는가 (증거 필요)
- [ ] 기존 테스트가 깨지지 않았는가
- [ ] 새 기능에 대한 테스트가 추가되었는가 (해당 시)

### 코드 품질
- [ ] 프로젝트 코딩 컨벤션을 준수하는가
- [ ] 주석/문서가 적절한가
- [ ] 불필요한 코드 / dead code가 없는가
- [ ] 보안 취약점 없는가 (입력 검증, 인증 등)

---

## 설계 작업 체크리스트

아키텍처/설계 문서에 추가 적용된다.

- [ ] 기술 결정의 근거가 명시되어 있는가
- [ ] 영향 범위가 식별되었는가
- [ ] 의존성이 명시되었는가
- [ ] 확장 포인트가 고려되었는가
- [ ] 트레이드오프가 설명되었는가

---

## 배포 작업 체크리스트

배포/운영 작업에 추가 적용된다.

- [ ] 배포 환경이 명시되었는가
- [ ] 롤백 방법이 확인되었는가
- [ ] 모니터링/로깅이 설정되었는가
- [ ] 배포 후 smoke test가 수행되었는가 (증거 필요)

---

## 판정 기준

| 결과 | 조건 |
|------|------|
| **PASS** | 해당 유형의 모든 필수 항목이 충족됨 |
| **CONDITIONAL PASS** | 사소한 항목 1–2개 미충족, 차후 수정 가능 |
| **FAIL** | 필수 항목 미충족 또는 Evidence 부재 |
