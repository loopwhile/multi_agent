# Meeting: Kickoff Meeting

## 목적

새 프로젝트 또는 주요 Work Order 시작 시 **목표, 스코프, 역할 배분, 초기 방향**을 정렬한다.

## 참석 역할

- another_me (진행)
- product_scope_agent
- system_architect_agent
- builder_agent (해당 시)

## 트리거

- 새 프로젝트 시작
- 대규모 Work Order 시작
- 프로젝트 방향 재설정

## 아젠다

### 1. 프로젝트/작업 개요 (5분)

- 배경과 목적
- 핵심 비즈니스 루프

### 2. 스코프 확인 (10분)

- 포함/제외 범위
- must-have vs nice-to-have
- 초기 수락 기준

### 3. 기술 방향 (10분)

- 기술 스택 / 아키텍처 방향
- 알려진 제약
- 기술 리스크

### 4. 역할 배분 & 일정 (5분)

- Work Order 담당 역할 배분
- 마일스톤 / 체크포인트 설정
- 다음 회의 시점

### 5. 오픈 이슈 (5분)

- 미결 사항 정리
- 즉시 결정할 것 vs 보류할 것

## 출력

```markdown
## Kickoff Minutes

### 날짜: YYYY-MM-DD
### 참석: {역할 목록}

### 합의 사항
- {합의 1}
- {합의 2}

### 결정 사항
- {결정 1} — 근거: {근거}

### 오픈 이슈
- {이슈 1} — 담당: {역할}

### Action Items
- [ ] {액션 1} — 담당: {역할} — 기한: {날짜}

### 다음 회의
- {일시 / 트리거 조건}
```
